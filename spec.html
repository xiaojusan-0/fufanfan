本轮修改内容

规则模板

  • 规则模板固定为 3 类：满减返、比例返、新人返。
  • 满减返展示“返金额”，比例返展示“返比例”，新人返展示“返金额”。
  • 满减返、比例返不存在“新客条件”，列表、详情、编辑弹窗均不展示该字段。
  • 新人返保留“新客条件”，支持“首次参与福利卡返利 / 首次在门店消费”。
  • 三个模板均展示“单日返利默认预算 ¥”，详情与编辑弹窗需与各自模板字段一致。

活动管理

  • 新增状态：草稿、待修改。
  • 草稿、待修改支持“查看、编辑”。
  • 已暂停活动新增“启用”按钮。
  • 查询条件新增玩法模板、操作人、活动开始日期、活动结束日期。
  • 列表新增玩法模板、操作人、开始日期、结束日期；列表不展示规则快照字段。

活动审核

  • 查询条件新增玩法模板、操作人、活动开始日期、活动结束日期。
  • 列表新增玩法模板、操作人字段。
  • 驳回弹窗去掉审核建议。

交易监控

  • 列表新增交易日期。
  • 查询条件新增交易日期，并支持重置查询。
  • 处理状态增加“已忽略”。
  • 异常状态新增“处理”按钮，点击跳转异常处理。
  • 命中交易操作列增加“快照”按钮。
  • 交易详情中的“规则快照”调整为“规则内容”。

异常处理

  • 列表新增交易日期、门店/商户字段。
  • 查询条件新增交易日期。
  • 已忽略状态不展示操作按钮。

门店管理、商户列表、退款/冲正

  • 门店管理全局去掉品类；导入方式只保留 Excel 导入；待完善状态对应营销状态为“不允许”。
  • 商户列表去掉品类，新增钱包余额，开通账号直接提示成功，不弹窗。
  • 退款/冲正新增交易日期及对应筛选，并增加重置按钮。

1. 产品定位

福返返是面向福利卡消费场景的业务营销一体化平台，用于承接交易同步、商户与门店基础资料、营销活动配置、规则判断、返利跟踪、异常处理和退款 / 冲正查询。

福返返负责福利卡消费后的业务处理、营销活动配置、规则内容留存、返利状态跟踪和运营异常处理。

2. MVP 范围

模块本期包含
总览展示今日交易、进行中活动、异常数量、核心业务流程和运营看板。
交易监控支持按交易、商户、门店、营销结果、资金状态、处理状态、交易日期查询；列表展示交易日期、资金状态、返利状态和处理状态；详情展示命中活动与规则内容。
商户列表商户基础资料来自商户中台同步；列表展示钱包余额；运营可在每条商户数据后开通平台账号，开通后直接提示成功。
门店管理支持查询、新增、编辑、Excel 导入门店；门店活动时间内允许编辑门店信息；待完善门店的营销状态为不允许。
规则模板维护满减返、比例返、新人返三类模板字段和模板默认值。
创建活动基于模板创建活动，字段从模板回填并允许调整；可用门店通过弹窗选择全部门店或部分门店。
活动管理支持按状态、玩法模板、操作人、活动日期筛选；支持查看、编辑、暂停、启用、作废等状态操作。
活动审核审核活动基础信息、玩法模板、操作人、活动日期、用户文案、预算与活动规则。
异常处理处理营销判断失败、资金失败、返利失败；列表展示交易日期和门店/商户，支持忽略。
退款 / 冲正作为结果查询页，支持交易日期筛选，状态仅为成功 / 失败。

3. 整体业务流程

用户福利卡消费
  → 发卡中台同步交易
  → 福返返接收并展示交易
  → 根据活动规则判断营销结果
  → 命中活动后校验商户营销金额并处理资金状态
  → 按返利时效给用户福利卡返利
  → 记录资金状态、返利状态、处理状态、命中活动和规则内容

5. 页面说明

5.1 总览

区域说明
指标卡展示今日交易、进行中活动、异常数量。
核心业务流程展示发卡同步交易、福返返处理、资金处理、返利发放。
运营看板展示待审核活动、异常处理等运营事项。

5.2 交易监控

类型内容
查询条件交易编号 / 商户 / 门店、营销结果、资金状态、处理状态、交易开始日期、交易结束日期。
列表字段交易编号、交易日期、商户/门店、金额、营销结果、应返、资金状态、返利状态、处理状态、操作。
操作详情、快照、处理。已命中交易展示快照；异常交易展示处理并跳转异常处理。
详情详情弹窗展示规则内容，不再使用“规则快照”作为字段名。

5.3 商户列表

字段 / 操作说明
查询条件商户名称 / 编号、营销状态、账号状态，支持重置查询。
列表字段商户名称、商户编号、门店数、钱包余额、营销状态、账号状态、操作；不展示品类。
开通账号点击后直接提示开通成功，不弹窗；成功后关闭按钮。

5.4 门店管理

能力说明
查询条件门店名称 / 终端号、商户、门店状态、营销状态，支持重置。
字段全局去掉品类。待完善状态对应营销状态为“不允许”。
编辑门店活动时间内允许编辑门店信息。启用/停用按钮移动到操作列。
导入只保留 Excel 导入；下载模板命名为“导入门店模板”；错误数据导入后为待完善；重复数据按“文件名+副本1、副本2...”命名；单次导入上限 100 条。

5.5 规则模板

模板展示字段说明
满减返满金额、返利方式、返金额、单人单日最高参与次数默认值、单人单日最高金额默认值、单日返利默认预算 ¥、返利时效、退款规则不展示新客条件。
比例返满金额、返利方式、返比例、单人单日最高参与次数默认值、单人单日最高金额默认值、单日返利默认预算 ¥、返利时效、退款规则不展示新客条件。
新人返满金额、返利方式、返金额、新客条件、单人单日最高参与次数默认值、单人单日最高金额默认值、单日返利默认预算 ¥、返利时效、退款规则新客条件支持首次参与福利卡返利、首次在门店消费。

5.6 创建活动

创建活动基于模板回填字段，运营可调整活动名称、玩法模板、返利方式、所属商户、可用门店、活动时间、满金额、返金额 / 返比例、参与限制、单日返利预算、活动总预算、预算耗尽动作和返利时效。

5.7 活动管理

能力说明
查询条件活动名称 / 商户、玩法模板、活动状态、操作人、活动开始日期、活动结束日期。
列表字段活动、商户、玩法模板、开始日期、结束日期、预算、操作人、状态、操作。
状态草稿、待审核、待修改、已上线、已暂停、已结束、已作废。
操作规则草稿/待修改支持查看、编辑；待审核/已上线支持查看、暂停、作废；已暂停支持查看、启用；已结束/已作废仅查看。

5.8 活动审核

区域说明
查询条件活动名称 / 所属商户、审核状态、玩法模板、操作人、活动开始日期、活动结束日期。
列表字段活动、所属商户、玩法模板、活动开始日期、活动结束日期、操作人、备注/原因、状态、操作。
操作待审核活动支持详情、通过、驳回；已通过、已驳回活动支持查看详情。
驳回驳回弹窗不展示审核建议。

5.9 异常处理

类型内容
查询条件交易编号 / 商户 / 门店、交易日期、异常类型、状态。
列表字段异常类型、交易编号、交易日期、门店/商户、原因、状态、操作。
操作营销判断失败支持重新判断、忽略；资金失败支持重试资金、忽略；返利失败支持重试返利、忽略；已忽略无操作按钮。

5.10 退款 / 冲正

查询条件列表字段状态
退款编号 / 原交易编号、商户、类型、状态、交易日期退款 / 冲正编号、原交易、交易日期、商户 / 门店、类型、退款金额、追回返利、状态、处理时间成功 / 失败

6. 关键口径

  • 规则模板维护默认值和模板边界；创建活动时作为活动字段回填。
  • 活动管理列表不展示规则快照字段，交易命中后在交易监控操作列查看快照。
  • 异常忽略视为人工关闭，已忽略状态不再展示操作按钮。
  • 门店待完善时营销状态为不允许。
  • 退款规则保留按比例追回、不追回两种。

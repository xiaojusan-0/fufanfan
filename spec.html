  • 查询条件新增玩法模板、操作人、活动开始日期、活动结束日期。

  • 驳回弹窗去掉审核建议。

  • 异常状态新增“处理”按钮，点击跳转异常处理。

  • 交易详情中的“规则快照”调整为“规则内容”。

  • 商户列表去掉品类，新增钱包余额，开通账号直接提示成功，不弹窗。
  • 退款/冲正新增交易日期及对应筛选，并增加重置按钮。

产品规则口径补充

  • 交易监控“营销结果”只展示已命中、未命中、未调用；已完成、异常、已忽略只归属处理状态。
  • 未达门槛、新客条件不满足、单人或单日上限超限等普通规则不满足，统一按未命中处理，不进入异常。
  • 营销判断失败仅用于规则快照缺失、活动配置异常、规则引擎异常、基础数据不一致等系统无法完成判断的情况。
  • 同一交易命中多个活动时，按用户本笔可获得返利金额最大选择活动；金额相同时按活动创建时间最早命中。
  • 单日预算不足但活动总预算充足时定义为尾单，允许占用次日预算并正常返利；活动总预算不足进入异常。
  • 活动总预算耗尽后活动自动暂停，后续不再参与匹配；商户充值营销金额后由运营手动上线。
  • 比例返金额按四舍五入保留到分；命中后立即占用预算，后续失败、忽略、退款或冲正再释放或追回。
  • 资金失败允许重试资金，返利失败允许重试返利；异常忽略后释放占用资金并放弃本轮返利。
  • 退款/冲正按活动规则配置追回；新人返的新客定义按活动规则配置取值。

1. 产品定位

福返返是面向福利卡消费场景的业务营销一体化平台，用于承接交易同步、商户与门店基础资料、营销活动配置、规则判断、返利跟踪、异常处理和退款 / 冲正查询。

福返返负责福利卡消费后的业务处理、营销活动配置、规则内容留存、返利状态跟踪和运营异常处理。

2. MVP 范围

退款 / 冲正作为结果查询页，支持交易日期筛选，状态仅为成功 / 失败。

3. 整体业务流程

用户福利卡消费
  → 发卡中台同步交易
  → 福返返接收并展示交易
  → 根据活动规则判断营销结果
  → 命中活动后校验商户营销金额并处理资金状态
  → 按返利时效给用户福利卡返利
  → 记录资金状态、返利状态、处理状态、命中活动和规则内容

6. 正式产品规则

6.1 交易状态字段

字段规则允许值
营销结果只表达营销判断结果，不表达流程处理结果。已命中、未命中、未调用
处理状态表达交易后处理流程结果。已完成、异常、已忽略
资金状态表达商户营销资金处理结果。成功、失败、无需
返利状态表达用户返利发放结果。成功、入账中、失败、无需

“已完成”“异常”“已忽略”“规则校验失败”“资金失败”“返利失败”不得展示在营销结果字段中。

6.2 普通未命中

未达到满额门槛、新客条件不满足、单人单日参与次数超限、单人单日返利金额超限、活动单日参与次数超限、活动单日返利金额超限、活动未开始、活动已结束、活动自动暂停后不参与匹配，均属于普通未命中，不进入异常处理。

字段展示值
营销结果未命中
应返金额¥0.00
资金状态无需
返利状态无需
处理状态已完成

6.3 营销判断失败

营销判断失败只用于系统无法完成营销判断，不用于普通业务未命中。

  • 属于营销判断失败：规则快照缺失或读取失败、活动配置数据异常、规则引擎执行异常、门店 / 商户 / 活动基础数据存在系统不一致、多活动冲突计算过程异常且无法得出最终命中活动。
  • 不属于营销判断失败：未达满额、非新客、超过单人上限、超过单日上限、活动未开始、活动已结束、活动自动暂停后不参与匹配。

营销判断失败进入异常处理，支持重新判断和忽略。

6.4 活动命中与冲突

  • 交易满足活动适用门店、活动时间、交易门槛、用户条件、限额规则和预算规则时，营销结果为已命中。
  • 同一交易同时满足多个活动时，按照用户本笔可获得返利金额最大命中活动。
  • 多个活动本笔可获得返利金额相同时，按照活动创建时间最早的活动命中。
  • 系统确定命中活动后，不再降级匹配其他活动。
  • 如果按冲突规则确定的第一个命中活动发生尾单余额不足，则按该活动的尾单规则处理，不改命中其他活动。

6.5 返利计算

返利方式计算规则
固定返利按照活动规则配置的满额门槛和返利金额计算。
比例返利按照活动规则配置的满额门槛和返利比例计算，计算结果按四舍五入保留到分。
 
比例返示例：
交易金额 100 元，返利比例 1.22%，应返金额 = 100 * 1.22% = ¥1.22
交易金额 1000 元，返利比例 1.22%，应返金额 = 1000 * 1.22% = ¥12.20

6.6 预算占用与尾单

  • 交易营销命中后立即占用活动预算。
  • 后续发生资金失败、返利失败、异常忽略、退款或冲正时，再按照对应规则释放、追回或调整预算。
  • 当前单日预算不足但活动总预算充足时，称为尾单；尾单允许占用次日预算，本笔交易仍正常返利。
  • 如果本笔命中活动的活动总预算不足，则进入异常处理，不降级命中其他活动。
  • 活动总预算耗尽后，活动自动暂停；自动暂停后不再参与后续交易匹配；商户充值营销金额后，由运营手动上线活动。
尾单示例：
活动单日预算 ¥500，当日已用 ¥495，本笔应返 ¥10，活动总预算仍充足。
处理结果：本笔正常命中，应返 ¥10；当日剩余 ¥5 被使用，额外 ¥5 占用次日预算。

6.7 异常处理

异常类型操作规则
营销判断失败重新判断、忽略用于系统无法完成营销判断。
资金失败重试资金、忽略资金失败允许重试资金。
返利失败重试返利、忽略返利失败允许重试返利。
活动总预算不足忽略或按异常处理流程处理活动总预算不足时不降级命中其他活动。

异常被运营点击忽略后，处理状态变更为已忽略，已占用资金释放，本轮返利放弃，后续不再自动重试，并保留操作人、操作时间和忽略原因。

6.8 退款 / 冲正

退款或冲正发生时，已发返利按照活动规则配置执行追回。活动规则配置“按比例追回”则按比例追回；配置“不追回”则不追回。退款 / 冲正状态展示本轮不调整，维持当前系统规则。

6.9 新人返

新人返的新客定义按照活动规则配置取值，可配置为首次参与福利卡返利、首次在商户消费或首次在门店消费。交易是否满足新客条件，以命中活动的规则配置为准。

6.10 规则模板与创建活动

  • 规则模板用于维护活动创建时的默认值和模板边界。
  • 规则模板字段可以展示“默认值”。
  • 创建活动时，字段由所选规则模板回填。
  • 创建活动页字段名称不展示“默认值”，可在字段占位提示中展示“模板回填，可调整”。
  • 活动创建完成并上线后，交易判断以活动规则配置为准；规则模板只作为创建活动时的回填来源，不直接参与已上线活动的交易判断。

6.11 验收口径

  • 交易监控营销结果不得混入处理状态。
  • 普通业务规则不满足不得进入异常。
  • 营销判断失败只用于系统无法完成判断。
  • 多活动冲突必须按本笔可获得返利金额最大命中；金额相同时按最早创建活动命中。
  • 尾单允许占用次日预算并正常返利。
  • 活动总预算不足进入异常；活动总预算耗尽后自动暂停。
  • 比例返金额必须四舍五入到分。
  • 命中后必须立即占用预算。
  • 忽略异常后必须释放占用资金并放弃本轮返利。
  • 资金失败允许重试资金，返利失败允许重试返利。
  • 退款 / 冲正必须按活动规则配置追回。
  • 新人返新客定义必须按活动规则配置取值。

7. 关键口径

  • 规则模板维护默认值和模板边界；创建活动时作为活动字段回填。
